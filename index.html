04 團隊合作 AI 工作流建議 — iOS / Android / Backend / Designer / QA / PM 之間怎麼用 AI 協作

馬上能落地的合作模式

  • 共用 /recap 格式做 daily standup：所有人用同一個 prompt 產進度摘要貼進 Slack #team-daily — PM/RD/QA 看到結構化資訊一致。
  • PR 自動跑 /code-review：每支 PR 推上去後 webhook 觸發，AI 先給 review，reviewer 從「找問題」變「審判 AI 找的問題」，瓶頸不在人。
  • team-skills repo 共建：你已經把 team-ai-changelog 上架了，鼓勵每位同事把自己常用 prompt / skill 也推上去，每週看誰新增了什麼。

建立團隊規範與共用資源

  • 定義 AI 介入邊界：用 hooks 列出禁止 AI 改的檔案（Secrets.swift、DB Migration、prod config），寫成 plugin 內 hook 全團隊套用。
  • 跨角色 skill 互相產出：QA 寫 test-case skill 給 RD 用、Designer 寫 design-token skill 同步三平台、PM 寫 stakeholder-recap skill — 每個 skill 標清楚適用角色。
  • PR review 三層分工：AI（/code-review）→ Peer（同 function）→ Senior（架構決策）。每層各專注，不重複工。

長期演進：把 AI 變團隊基建

  • 內網 exec-server / plugin marketplace：codex exec-server 或 Claude plugin 部署在內網 build agent，CI 不用 spawn 新 process，全團隊共享同一份規則。
  • AI 工作流 retro：每 Sprint retro 花 10 分鐘看「哪些 prompt 真有產出、哪些只是噪音」，淘汰 / 改寫 skill 與 hook。把 AI 使用當成有壽命的資產。
  • 量化團隊 AI 使用：用 OTEL + status line 收集 effort 使用、plugin 命中率、subagent 完工率 — 資料驅動規範，不是憑感覺。

04 設計師怎麼用 — 把 Claude Design 接進 設計流程

馬上能試

  • 從既有 repo 萃取設計系統：連 行動銀行 App 前端 repo，讓 Claude 讀實際元件產出與程式一致的 token / 元件雛形。
  • 對話式產原型：用 chat + 調整滑桿（palette／間距／動畫）快速試版面方向，先看一版再微調。

交付與協作

  • 一鍵 Handoff 給 Claude Code：原型直接變成帶設計脈絡的編碼 session，RD 接著做而非看圖猜。
  • 匯出多格式：Canva／PDF／PPTX／HTML，對內提案或交付都方便。

注意

  • 研究預覽：功能與輸出仍在演進，正式專案前先小範圍驗證。
  • 資安：連結 repo 前確認符合 本行程式碼外連規範。

04 設計師怎麼用 — 把近期 Figma 更新接進交付流程

交付品質把關

  • Check designs：handoff 前對 file 跑一次，把寫死值換成 token、修對比違規、抓 detached component。
  • Slot 設定：用最小／最大圖層、僅允許偏好實例，讓共用元件更難被誤用。

探索與參考

  • Chrome 擴充抓網頁成圖層：競品 / 既有頁面帶進畫布二創，不用重畫。
  • Figma Make Plan mode：複雜版面先讓 Make 擬計畫再生成，結果更貼需求。

團隊面

  • 桌面 Tab Groups：用顏色／分組整理多專案檔案，減少找檔時間。
  • AI credits：留意 Make／AI 功能用量，必要時按量加購。
